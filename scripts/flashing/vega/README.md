# Milk-V Vega flashing kit

Recover and reflash the Milk-V Vega over JTAG with a patched Nuclei OpenOCD.
The kit lives in this OpenWrt port so a fresh checkout can build the exact
OpenOCD used for NOR recovery and NAND image updates.

## Layout

| Path | Purpose |
|---|---|
| `bootstrap.sh` | Clone Nuclei `riscv-openocd`, apply patches, build OpenOCD, and generate NAND stubs. |
| `patches/01-nuspi-burst-write.patch` | NUSPI slow-path: drop per-byte FIFO poll. |
| `patches/02-riscv013-burst-helper.patch` | RV0.13 driver: bulk DMI writes via `abstractauto`. |
| `configs/openocd-board.cfg` | Source copy of the FTDI pinout + `work-area-size 0` config installed as `riscv-openocd/openocd-slow.cfg`. |
| `flash.sh` | Flash NOR SPL, NAND kernel, or NAND rootfs. |
| `dump-flash.sh` | Read SPI NOR back to a file. |
| `jtag-dump.sh` | Capture a compact CPU/register snapshot. |
| `flash-region.tcl` | Unlock, erase, write, verify a NOR sector range. |
| `unlock-and-erase.tcl` | Status-register unlock + erase. |
| `_common.tcl` | NUSPI register helpers, `w25q_unlock`. |
| `stubs/` | Source for tiny SPI-NAND helpers generated by `bootstrap.sh`. |

## Build

```sh
./scripts/flashing/vega/bootstrap.sh
```

`bootstrap.sh` clones `riscv-openocd` into `scripts/flashing/vega/riscv-openocd`,
checks out the pinned Nuclei commit, applies `scripts/flashing/vega/patches/*.patch`,
copies the board config to `riscv-openocd/openocd-slow.cfg`, builds
`src/openocd`, and writes generated NAND helper binaries under
`scripts/flashing/vega/stubs/`. Set `SKIP_DEPS=1` if build dependencies are
already installed and you do not want the script to call Homebrew or `apt-get`.

## Flash OpenWrt

From the repository root after `./build.sh`:

```sh
OUT=openwrt/bin/targets/milkv_vega/milkv_vega

./scripts/flashing/vega/flash.sh nor "$OUT/vega-spl.bin"
./scripts/flashing/vega/flash.sh nand kernel "$OUT/openwrt-milkv_vega-milkv_vega-milkv_vega-ubifs-kernel.bin"
./scripts/flashing/vega/flash.sh nand rootfs "$OUT/openwrt-milkv_vega-milkv_vega-milkv_vega-ubifs-rootfs.ubi"
```

The scripts default to the OpenOCD and `openocd-slow.cfg` produced by
`bootstrap.sh`. The explicit form, matching the old workflow, is:

```sh
OPENOCD=$PWD/scripts/flashing/vega/riscv-openocd/src/openocd \
OPENOCD_CFG=$PWD/scripts/flashing/vega/riscv-openocd/openocd-slow.cfg \
./scripts/flashing/vega/flash.sh nor /path/to/vega-spl.bin
```

The NOR file's offset 0 maps to chip offset 0. `flash.sh nor` computes the
sector range from the file size, erases those sectors, writes the file, and
spot-verifies each sector.

## Backup and triage

```sh
./scripts/flashing/vega/dump-flash.sh /tmp/vega-nor.bin 0x400000
./scripts/flashing/vega/jtag-dump.sh
```

`sector-diff.sh` is available for comparing equal-layout NOR images and prints
the 64 KiB sector numbers that differ.

## UART

Console on the second FTDI channel, 115200-8N1. macOS device:
`/dev/cu.usbserial-*`.

```sh
> /tmp/uart.log
python3 - /dev/cu.usbserial-XXXX 115200 /tmp/uart.log <<'PY' &
import serial, sys
s = serial.Serial(sys.argv[1], int(sys.argv[2]), timeout=0.1)
with open(sys.argv[3], 'wb') as f:
    while True:
        b = s.read(256)
        if b: f.write(b); f.flush()
PY
# power-cycle, then:
LC_ALL=C tr -d -c '[:print:]\n\r\t' < /tmp/uart.log
```

## Recovery details

- **Basic recovery.** Power-cycle the board, run `bootstrap.sh`, flash
  `vega-spl.bin` to NOR, power-cycle again, then flash the NAND kernel and
  rootfs if the existing NAND contents are stale or missing.
- **Slow path.** `work-area-size 0` tells `nuspi.c` to skip the ILM
  algorithm and write the NUSPI registers (`TXDATA`/`FCTRL`/`CSMODE` at
  `0x10014000`) directly over JTAG/DMI.
- **Status-register unlock.** Fresh W25Q32JV reports `SR1=0xC0`, `SR2=0x7E`;
  all programs are NAKed. Volatile `WREN` (`0x50`) + `WRSR` (`0x01 0x00 0x00`)
  + `WRSR2` (`0x31 0x00`) clears it for the session.
- **Burst writes.** Patch 1 removes the per-byte FIFO-full read. Patch 2 adds
  `riscv013_write_burst_to_addr()` which loads `sw s1, 0(s0); ebreak` into
  progbuf, enables `DM_ABSTRACTAUTO.autoexecdata`, and queues N×`DM_DATA0`
  writes per `riscv_batch` — one USB round-trip per batch instead of per byte.
  ≈ 9 KB/s → ≈ 85 KB/s.

## Gotchas

- JTAG ≤ 8 MHz. Above ~20 MHz the DMI bus corrupts.
- Status-register unlock is per-power-cycle.
- Reading `0x40000000+` before DDR init can wedge the AXI bus; only power cycle
  recovers.
